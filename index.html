如何高效地读数学书

发表于 2024-02-18 分类于 数学

1. 面向应试读书

1.1 基本原则

  • 应试的目的：通过考试或者考试拿高分。

  • 对于应试，学的过多或过少都是不可取的，应该针对考试的范围来学习。

  • 为了应试所学的知识在考后很容易遗忘，必须要及时复习（为了应试而学的知识也是知识，遗忘就太可惜了）。

  • 对于比较难的定理的证明，可以先跳过，先知道怎么用这个定理，等到后面积累了更多的经验和例子以后再回来看（但是一些基本性质和基本定理的证明在第一次看的时候还是要掌握的）。

1.2 准备

  • 开始学习前需要确定主要的参考书和要看的章节。

1.3 知识的整理

  • 在读完一节/一章后重新再读一遍，用自己的方式总结概要和整理重点，记录所有的定义、性质和重要定理。

  • 很显然的证明不需要整理；简单性质或定理的证明就记关键一步或关键思想（让自己看到以后就能知道怎么证），不需要全记；大定理的证明可以先不记，如果能读懂大定理的证明，就记关键的几步。

  • 可以把自己的笔记分享出来。

1.4 复习

  • 最佳的复习时间是考试后，在考试后把知识重新梳理一遍，会有新的理解，尝试问自己以下几个问题：为什么要这样定义？这个概念有什么用？这个定理可以怎么用？这个定理中的条件在证明的哪一步用到了？如果改变条件，定理的结论会有什么变化？这门课和之前的课程有什么联系？

2. 面向积累读书

2.1 基本原则

  • 明确动机：为什么读书？为谁而读书？要读到什么程度？要读完吗？什么时候要读完？读完以后可以做什么？现在有能力读完吗？

  • 明确能力：数学知识可以分为两类——静下心来花点时间就能读懂的和怎么努力都读不懂的。对于前者，静下心来努力去学就可以，不要有进度焦虑；对于后者，及时放弃，换一本自己看得懂的书（读书归根结底是为了学知识，如果一昧地追求所谓的好书或有着高观点的书，而没能学到知识或者说学得很痛苦，就本末倒置了）。

  • 不必抗拒承认自身能力的不足，能力总会提高的。

  • 它山之石可以攻玉，不要局限在一本书/一个学科，广泛涉猎，同一个知识点会有不同的讲法，同一个定理也会有不同的证法，体会其中的细微差异有助于提高自身水平。

  • 既然选择了学数学，就要坚持到一个终点，如果实在是学不下去，再放弃也未尝不可。

2.2 找到适合自己的节奏和感觉

  • 读书学习的好处是可以自行控制节奏，知识点忘记了可以翻阅查找。

  • 好的读书节奏：读书读到“爽”。

  • 读书要讲求效率，一天读几十页，并且反复读，才是高效的读书方法。每次读都会有新的体会，能加深理解，也能避免遗忘。

  • 读书要达到心流的状态，可以通过实验找到实现心流的方法。

  • 读书的关键在于享受读书的过程，对于长期学习，良好的心理状态至关重要。

2.3 整理知识：

  • 整理笔记（方法和应试情形下做笔记的方法类似），还可以把自己对某个知识点或某个问题的理解写成专题文章。

  • 构建起知识点之间的联系（可以利用卡片盒笔记法）。

  • 尝试和志同道合的人进行高质量的交流（讨论定理、公式、论文等），并且交流一定要有成果，例如对一个定理的新的理解方式或者是分析出论文的创新点。

如何搭建Python虚拟环境

发表于 2024-01-21 更新于 2024-01-26 分类于 计算机科学与技术

1. 前言

在使用Python进行开发时，我们经常会遇到以下两个问题：

1.由于误操作，导致全局Python（依赖）环境被破坏，无法运行。

2.某个包无法在当前的Python版本下运行。

对于Linux而言，这两个问题尤为棘手。大多数Linux发行版都自带Python，并且一些系统功能的实现也依赖于Python环境，随意改动系统自带的Python环境，有可能会导致系统无法正常运行。

解决这两个问题最有效的方法是搭建一个Python虚拟环境。本文介绍两种方法：使用Python自带的venv模块和使用Python环境管理工具pyenv，前者可以快速地搭建一个轻量级的Python虚拟环境，后者则可以在一个系统中安装多个版本的Python，并为某个目录指定该目录下运行的Python版本。

注：

本文使用的Linux发行版为Ubuntu。其他发行版用户请自行将文中所用的包管理器替换为所用发行版对应的包管理器。

2. Python虚拟环境简介

Python虚拟环境指与系统Python环境隔离的Python环境。每个Python虚拟环境都拥有完全独立的包和依赖关系，即使当前的Python虚拟环境被破坏，其他虚拟环境和系统环境仍能正常运行。

由于不同项目所需的Python版本、包和依赖关系不同，因此推荐为每个项目都单独设立一个Python虚拟环境，并且在虚拟环境中安装包，而不是在系统环境中安装包。

注：

要进一步了解Python虚拟环境，可以访问：PEP 405 – Python Virtual Environments。

4. 使用pyenv搭建Python虚拟环境

使用venv模块虽然能方便快捷地搭建Python虚拟环境，但这种方法有一个致命缺陷，就是所搭建的虚拟环境中的Python版本与系统中的Python版本一致，对于本文开头提到的某个包无法在当前Python版本下运行的问题，用venv就无法解决。

要搭建Python版本与系统Python版本不一致的虚拟环境，就要用到名为pyenv的Python环境管理工具。通过pyenv，我们就可以在一个系统中安装多个Python版本，并且可以为某个目录指定在该目录及其子目录下运行的Python版本。

C++专题：C++与交互式终端中的换行符

发表于 2024-01-21 更新于 2024-01-22 分类于 计算机科学与技术

在学习C++时，我们偶尔会意外写出死循环，导致这种错误的原因有时是因为我们写的代码有BUG，有时则是因为我们忽略了一些C++的特性。对于前者，我们很容易就能解决，而对于后者，如果不能弄清楚我们忽略了哪些特性，就会非常棘手。本文将介绍C++中一种常见的死循环错误：由C++程序不能识别用户在交互式终端中输入的换行符所导致的死循环。

1. 问题描述

《C++ Primer》（第五版）中的练习3.17要求读者写一个C++程序，其功能是从标准输入中读取一组单词，并将其存入到一个向量中，然后将单词的所有字母都改为大写，最后将改写后的单词输出。程序代码如下所示：

在这里先不讨论具体的技术细节，让我们把目光聚焦在这个程序需要解决的一个关键问题上——因为用户输入的单词数量是不确定的，所以程序需要持续读取用户的输入，并在输入完成后执行后续的代码。《C++ Primer》的1.4.3节提供了对该问题的解决方案：通过将输入流作为while循环条件的方式，就能持续读取数量不定的数据。原因在于，当一个输入流作为条件时，条件实际上是在检测流的状态，若流没有错误，则条件为真。当遇到文件结束符或无效输入时，流的状态就会变为无效，此时条件为假，循环结束，程序将执行后续的代码。上述程序中的while循环

换行符（\n）就是一种文件结束符，因此从理论上来说，当用户在终端中输入完单词并按下回车键（输入换行符）后，程序就会执行后续代码，并输出结果。然而实际情况却并非如此，当我们在终端中输入完单词并按下回车键后，程序并没有输出结果，而只是跳转到了下一行：

2. 问题的原因和解决

导致上述while循环成为死循环的根本原因是：C++程序不会将交互式终端（interactive terminal）中输入的换行符识别为文件结束符。 在用户按下回车后，输入流仍在等待文件结束符，此时条件将一直为真，while循环就成为了死循环。

解决这个问题的方法也很简单，既然在终端中输入的换行符不会被识别为文件结束符，那就不在终端中以交互的方式来运行程序，而是事先将输入写入到一个文件中，然后运行程序，并将输出结果保存在另一个文件中，即以文件重定向的方式来运行程序。Linux中文件重定向的命令为：
