如何搭建Python虚拟环境

发表于 2024-01-21 更新于 2024-01-22 分类于 计算机科学与技术

1. 前言

在使用Python进行开发时，我们经常会遇到以下两个问题：

1.由于误操作，导致全局Python（依赖）环境被破坏，无法运行。

2.某个包无法在当前的Python版本下运行。

对于Linux而言，这两个问题尤为棘手。大多数Linux发行版都自带Python，并且一些系统功能的实现也依赖于Python环境，随意改动系统自带的Python环境，有可能会导致系统无法正常运行。

解决这两个问题最有效的方法是搭建一个Python虚拟环境。本文介绍两种方法：使用Python自带的venv模块和使用Python环境管理工具pyenv，前者可以快速地搭建一个轻量级的Python虚拟环境，后者则可以在一个系统中安装多个版本的Python，并为某个目录指定该目录下运行的Python版本。

注：

本文使用的Linux发行版为Ubuntu。其他发行版用户请自行将文中所用的包管理器替换为所用发行版对应的包管理器。

2. Python虚拟环境简介

Python虚拟环境指与系统Python环境隔离的Python环境。每个Python虚拟环境都拥有完全独立的包和依赖关系，即使当前的Python虚拟环境被破坏，其他虚拟环境和系统环境仍能正常运行。

由于不同项目所需的Python版本、包和依赖关系不同，因此推荐为每个项目都单独设立一个Python虚拟环境，并且在虚拟环境中安装包，而不是在系统环境中安装包。

4. 使用pyenv搭建Python虚拟环境

使用venv模块虽然能方便快捷地搭建Python虚拟环境，但这种方法有一个致命缺陷，就是所搭建的虚拟环境中的Python版本与系统中的Python版本一致，对于本文开头提到的某个包无法在当前Python版本下运行的问题，用venv就无法解决。

本章介绍了C++中的变量和内置类型，并简单讲解了如何自定义数据结构。

2.1 基本内置类型

C++的基本内置类型包括算术类型（arithmetic type）和空类型（void）两大类。其中算术类型包含了字符、整型数、布尔值和浮点数。空类型不对应任何具体值，仅在一些特殊情形下使用，例如，当函数不返回任何值时，就应使用空类型作为其返回类型。

2.1.1 算术类型

算术类型分为两类：整型和浮点型。整型包含整数、字符和布尔值，浮点型即浮点数。

各种数据类型的尺寸（即该类型数据所占的比特数）在不同机器上往往有所差别。数据类型所占的比特数不同，它能表示的数据范围就不同。

下表列出了C++中各算术类型的含义及其最小尺寸。C++仅对算术类型的最小尺寸作出规定，同时允许编译器为算术类型赋予更大的尺寸。

计算机如何存储数据

计算机以比特序列存储数据，每个比特非0即1。例如：

0001101101110001011001000011011 ...

大多数计算机以2的整数次幂个比特作为块来处理内存，可寻址的最小内存块称为字节（byte）。存储的基本单元是字（word），一个字通常由几个字节组成。大多数计算机的字节由8比特组成，字则由32或64比特组成，即4或8字节。

计算机会将内存中的每个字节与一个被称为内存地址（address）的数字关联起来。通过内存地址，计算机程序就能对内存中的数据进行定位（寻址）和访问。为了赋予内存中某个地址以明确的含义，必须首先知道存储在该地址的数据的类型。数据类型决定了数据所占的比特数，以及如何解释这些比特的内容。

算术类型详解

布尔类型（bool）的取值为真（true）或假（false）。

C++中基本的字符类型是char，一个char的空间应该确保能够存放机器基本字符集中任意字符所对应的比特序列，也就是说，一个char的大小就是1字节。其他字符类型则用于拓展字符集。wchar_t用于存放机器最大扩展字符集中的字符。char16_t和char32_t则用于存放Unicode字符集中的字符（Unicode字符集是用于表示所有自然语言中字符的标准）。

除布尔类型和字符类型外，其他整型用于表示不同尺寸的整数。

浮点型用于表示单精度、双精度和扩展精度值。通常float以1个字（32比特）来表示，double以2个字（64比特）来表示，long double以3或4个字（96或128比特）来表示。一般来说，float和double分别有7和16个有效位，long double常用于特殊的硬件，其有效位依赖于具体的实现方式。

注：

由于float和double所占的比特数不同，因此二者能表示的数值的范围（即有效的位数）就不同。

带符号类型和无符号类型

除去布尔类型和扩展的字符类型外，其他整型可以划分为带符号（signed）和无符号（unsigned）两种。带符号类型可表示大于0、等于0和小于0的值，而无符号类型只能表示大于或等于0的值。

类型int、short、long和long long都是带符号的。通过在类型名前加上unsigned就可以得到对应的无符号类型，例如unsigned long，并且unsigned int可缩写为unsigned。

char被分为三种：char、signed char和unsigned char。但实际上char只有带符号和无符号两种表现形式，且char是带符号还是无符号由编译器决定。

无符号类型的所有比特都用于存储数据，而带符号类型的第一个比特会被用于表示符号。因此，二者能表示的数据范围不同，带符号类型在正向上表示的范围比无符号类型少一半，而少掉的这一半会在负向上表示出来。

注：

1.在一些特殊情形下，为节省内存空间，可以用char表示小范围的8比特整数（char仅占8比特，而int最小也要占16比特）。unsigned char可表示0至255范围内的整数，signed char可表示-128至127范围内的整数。

2.选择数据类型的一般标准：

  • 当明确数值不可能为负时，选用无符号类型。

  • 使用int执行整数运算。

  • 在算术表达式中不要使用char和bool，仅在保存字符或布尔值才使用它们。如果要用char来表示数值不大的整数，则应指定是signed char还是unsigned char。

2.2 变量

变量提供一个具名的、可供程序操作的存储空间。C++中的每个变量都有其数据类型，数据类型决定了变量所占内存空间的大小和布局方式、该空间能存储的值的范围，以及变量能参与的运算。一般来说，变量（variable）和对象（object）这两个术语可以互换使用。

2.2.1 变量定义

变量定义的基本形式是：首先是类型说明符（type specifier），随后紧跟由一个或多个变量名组成的列表，其中变量名间以逗号分隔，最后以分号结束。列表中每个变量的类型都由类型说明符指定，定义时还可以为一个或多个变量赋初始值。

注：

对象通常指一块能存储数据且具有某种数据类型的内存空间。不难看出，对象和变量的概念很相似，因此二者可以互换使用。

初始值

初始化（initialized）： 对象在创建时获得一个初始值。

既可以用字面值初始化变量，也可以用表达式初始化变量。并且当一次定义了两个或多个变量时，可以用先定义的变量的值去初始化后定义的变量。

注：

1.当在一条语句中定义多个变量时，变量的定义有先后顺序，且若先定义的变量创建失败，则后面定义的变量都不会被创建。具体说来，若在一条语句中先后定义了a、b、c三个变量，且在后面调用了b和c，若a创建失败，则编译器不仅会报告a创建失败，还会报告b和c未声明。

2.初始化和赋值是两个完全不同的操作。初始化是在创建对象时为其赋予一个初始值，而赋值是将对象的当前值擦除并赋予一个新值。例如，2.4节将介绍，可以对常量初始化，但不能对常量赋值。

2.2.2 变量声明和定义的关系

为了允许将程序拆分成多个逻辑部分来编写，C++支持分离式编译（separate compilation）机制，该机制允许将程序源代码分割成若干个文件，每个文件可被独立编译。

如果要将程序源代码分为若干个文件，就必须要有在文件间共享代码的方法。例如，一个文件可能会使用另一个文件中定义的变量。常见的例子是，cin和cout定义在iostream标准库头文件中，但可以被我们所写的程序使用。

为了实现分离式编译，C++将变量的声明和定义区分开来。声明（declaration）告知编译器存在程序元素或名称，一个文件如果想使用在另一个文件中定义的名称，就必须要对该名称进行声明。而定义（definition）会创建与名称相关的实体。

声明和定义的联系是：

1.声明和定义都会指定变量的数据类型和名称。

2.定义是声明的一种，定义变量的同时也会声明变量。

1.定义会申请存储空间，而声明不会申请存储空间。

注：

C++是一种静态类型（statically typed）语言，其含义是变量的类型在编译时确定，且编译器会进行类型检查（type checking），若编译器发现某个变量的类型不支持其参与的操作，则编译器会报错。静态类型语言的变量的类型必须在声明时确定，并且类型确定后一般不能改变。

与静态类型语言对应的是动态类型（dynamically typed）语言，其含义是变量的类型在运行时根据赋给变量的值自动确定。变量在声明时不需要明确类型，且变量的类型可以改变。

6.1.1将要提到，函数体内定义的变量一般没有固定地址，因此constexpr指针不能指向这样的变量。相反地，定义在所有函数体之外的对象其地址值不变，因而其地址能用来初始化constexpr指针。在6.1.1节中还将提到允许函数定义一类有效范围超出函数本身的变量，这种变量和定义在函数体之外的变量一样也有固定地址。因此，constexpr引用能绑定到这样的变量上，constexpr指针也能指向这样的变量。

指针和constexpr

使用constexpr定义的指针是常量指针。并且和其他常量指针类似，constexpr仅限定了指针本身是常量，而对于指针所指的对象是不是常量则没有作出限定。

2.6.3 编写自己的头文件

尽管如19.7节所讲可以在函数体内定义类，但这样的类会受到限制，因此一般不在函数体内定义类。并且，如果要在多个文件中使用同一个类，则类的定义必须保持一致。

基于上述原因，类一般定义在头文件中，且头文件的名字应与类的名字相同。如前所述，Sales_data类定义在头文件Sales_data.h中。

头文件通常包含那些只能被定义一次的实体，如类、const和constexpr变量等。头文件也经常包含其他头文件的功能。例如，因为Sales_data类中定义了名为bookNo的string成员，所以需要在Sales_data.h中包含string头文件。

预处理器概述

有时，我们会（意外地）多次包含同一个头文件。确保程序在头文件被重复包含后仍能正常工作的常用技术是预处理器（preprocessor），它由C++语言从C语言继承而来。预处理器是在编译前执行的一段程序，可以部分地改变我们编写的代码。我们已经用到了#include这一项预处理功能，预处理器在遇到#include后，会用指定的头文件的内容替换#include。

为了确保程序在头文件被重复包含后仍能正常工作，我们需要使用名为头文件保护符（header guard）的预处理功能。头文件保护符依赖于预处理变量，而预处理变量有两种状态：已定义和未定义。#define指令把一个名字定义成预处理变量，#ifdef和#ifndef指令则负责检查预处理变量的状态。#ifdef当且仅当预处理变量已定义时为真，#ifndef当且仅当预处理变量未定义时为真。一旦检查结果为真，预处理器将执行后续代码直至遇到#endif。

命名空间

注意到先前在使用标准输出cout时，没有直接写cout，而是写了std::cout。前缀std::指出使用的名字cout是定义在名为std的命名空间（namespace）中的。

命名空间是C++的一个基本特性。C语言规定变量名不能和关键字冲突，但没有规定变量名不能和库中的变量名冲突。为了避免这种冲突，C++引入了命名空间，用于指定所使用名称的出处。标准库定义的所有名字都在命名空间std中。

在使用标准库中的名字时，必须显式地指出使用的名字来自于命名空间std。例如，要使用标准输出cout，就必须写std::cout，其中作用域运算符::的功能就是指定使用的名字cout来自命名空间std。

1.3 注释

注释（comments）的作用是帮助人类理解代码，通常用于概述算法，说明变量的用途和解释晦涩难懂的代码段。编译器在编译时会忽略掉注释。

C++中注释的种类

C++中的注释有两种。

单行注释： 以双斜线//开始，以换行符结束。单行注释中可以包含任何文本，包括额外的双斜线。

多行注释： 以/*开始，以*/结束（/*和*/称为注释界定符），其中可以包含除/*和*/以外的任意内容，包括换行符。编译器会将/*和*/之间的内容都当作注释。

在使用多行注释时，最好指明多行注释中的程序行是注释的一部分，常用的方法是在多行注释中每一行的开头写一个星号*。

多行注释不能嵌套

一个多行注释不能嵌套在另一个多行注释中，否则编译器会报错。这是因为多行注释是以/*开始，以*/结束的，如果将一个多行注释嵌套在另一个多行注释中，那么编译器就无法理解注释从哪里开始，到哪里结束。若编译下面的代码，则编译器会报错：
